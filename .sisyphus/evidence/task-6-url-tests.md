### 1. Slugify Function Test

**Input**: `"Lomba Fashion Show Kebaya & Pakaian Adat - Happy Play Kids"`

**Expected Output**: `lomba-fashion-show-kebaya-pakaian-adat-happy-play-kids`

**Actual Output**: `lombafashionshowkebayapakaianadat-happyplaykids`

**Status**: ❌ MISMATCH

### 2. Root Cause Analysis

The slugify function in `src/utils/supabaseApi.ts:501-503`:

```typescript
function slugify(text: string): string {
  return text.toLowerCase().replace(/[^a-z0-9\s-]/g, '').replace(/\s+/g, '-').replace(/-+/g, '-').trim();
}
```

**Problem**: The regex `/[^a-z0-9\s-]/g` removes ALL non-alphanumeric characters except spaces and hyphens. However, when processing "Pakaian Adat - Happy Play Kids":

1. First regex: Removes `&` → "Lomba Fashion Show Kebaya  Pakaian Adat - Happy Play Kids"
2. Second regex: Replaces spaces with hyphens → "lomba-fashion-show-kebaya--pakaian-adat---happy-play-kids"
3. Third regex: Collapses multiple hyphens → "lomba-fashion-show-kebaya-pakaian-adat-happy-play-kids"

**BUT** the actual output shows spaces are being removed entirely in some cases, resulting in:
`lombafashionshowkebayapakaianadat-happyplaykids`

This suggests the regex is not preserving spaces correctly before the hyphen replacement step.
